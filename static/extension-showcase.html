Project framing

What changed from the original app

From Flask app to packaged extension

The original project was a Flask visualizer. For Chrome Web Store publishing, the solving logic was ported to bundled JavaScript so the extension can run locally and stay compliant with Manifest V3 rules on remote hosted code.

One codebase, three surfaces

The workspace now runs as a Chromium extension, as a hosted demo through the Flask app, and as a portfolio case study with public privacy and publication material.

Review-friendly design

The package requests only local storage, uses no host permissions, and does not inject code into websites. That makes the intent narrow, easier to explain, and safer to review.

Technical choices

Architecture highlights

Local solver engine

The Python N-Queens and Sudoku logic was translated into a browser-ready solver module, keeping the extension self-contained for store review.

Worker-based computation

Long-running steps are offloaded to a Web Worker, so the interface remains responsive during solving and benchmarking.

Portfolio-ready routing

The Flask app now exposes clean routes for the extension workspace, popup preview, case-study page, and privacy policy.

Publication assets

The repository now includes packaging scripts, generated icons, store listing copy, and a privacy policy draft.
